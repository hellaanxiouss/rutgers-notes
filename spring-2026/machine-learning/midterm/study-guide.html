Lecture 1
The Learning Problem

What is Machine Learning?

ML = improving some performance measure with experience computed from data. It is used when a rule cannot be explicitly programmed — e.g., recognizing faces, filtering spam.

When Should We Use ML?

  • When the pattern exists (there IS something to learn)
  • When we cannot define the rule mathematically (no explicit formula)
  • When we have data on the pattern

ML vs. Related Fields

  • ML vs. AI: ML is one route to AI (others: game trees, expert systems)
  • ML vs. Statistics: Stats focuses on math proofs, ML focuses on computation
  • ML vs. Data Mining: DM finds interesting properties, ML approximates f
📝 Exam Tip: Be ready to explain WHY ML is used instead of writing explicit rules. Also practice mapping spam detection, credit approval, face recognition onto X, Y, f, D, H, A, g.

Lecture 2
Learning to Answer Yes/No — Perceptron & PLA

The Perceptron Hypothesis

A perceptron computes a weighted score of features and thresholds it. In 2D it's a line; in higher dimensions it's a hyperplane.

Perceptron Formulah(x) = sign(w₀ + w₁x₁ + ... + wdxd) = sign(wᵀx)

Perceptron Learning Algorithm (PLA)

  • Start with w₀ = 0 (or any vector)
  • Find a misclassified point (xₙ, yₙ) where sign(wᵀxₙ) ≠ yₙ
  • Update: w ← w + yₙ·xₙ (correct the mistake)
  • Repeat until no mistakes on training data
GuaranteePLA ONLY guaranteed to halt if data is linearly separable.
Max mistakes T ≤ (R/ρ)² where R = max data radius, ρ = margin

Pocket Algorithm (for non-separable data)

Keep the "best so far" weights (fewest training errors) in your "pocket". Run for a fixed number of iterations and return pocket weights.

  • Pros: works on noisy, non-separable data
  • Cons: slower than PLA (must check if new weights are better each round)
⚠️ Key Distinction: PLA = linearly separable data only. Pocket = noise-tolerant version for messy real data. This comparison is very likely to appear on the exam.

Types of Learning

By Output Space Y

  • Binary Classification: Y = {−1, +1} → spam/not spam, sick/not sick
  • Multiclass Classification: Y = {1,...,K} → digit recognition (0–9)
  • Regression: Y = ℝ → stock price, temperature, credit limit
  • Structured Learning: Y = complex structures → sentence parsing

By Data Label Type

  • Supervised: every xₙ comes with a label yₙ
  • Unsupervised: no labels — e.g., clustering, density estimation
  • Semi-supervised: some labels, many unlabeled
  • Reinforcement Learning: learn from reward/penalty signals (e.g., game agents)

By Protocol (How Data Arrives)

  • Batch learning: all data at once (most common)
  • Online learning: data arrives sequentially, model updates one at a time
  • Active learning: model asks for labels on confusing examples

By Input Space X

  • Concrete features: hand-crafted, informative (e.g., coin size and mass)
  • Raw features: need processing (e.g., pixel values)
  • Abstract features: not directly informative (e.g., user ID)
📝 Exam Tip: Know the 4 axes (Y, label type, protocol, X) and be able to classify a described problem on each axis.

Lecture 4
Feasibility of Learning

The Core Problem

Without assumptions, learning is impossible — there are always multiple valid target functions consistent with training data (the "No Free Lunch" intuition).

Hoeffding's Inequality — Probability to the Rescue

If we sample marbles from a bin, the fraction of orange marbles in our sample (ν) is close to the true fraction (μ) with high probability. This extends to learning: Ein (in-sample error) ≈ Eout (out-of-sample error) for large N.

Hoeffding's BoundP[|Ein(h) − Eout(h)| > ε] ≤ 2·exp(−2ε²N)

Works for a FIXED h. For multiple hypotheses → multiply by |H| (Union Bound)

The VC Bound

  • Eout(g) ≤ Ein(g) + complexity penalty
  • Penalty grows with model complexity (dVC) and shrinks with N
  • dVC = VC dimension = max points the model can shatter
  • Perceptron in ℝd has dVC = d + 1
  • Rule of thumb: need N ≥ 10 × dVC
📝 Exam Tip: You don't need to derive the VC bound — just explain what it says: more data = better generalization; more complex model = harder to generalize.

Lecture 5
Noise and Error

Probabilistic Target

Instead of a deterministic f(x), we have a target distribution P(y|x). The "ideal" label at x is the most likely class, and the rest is noise.

ExampleP(+1 | x) = 0.7, P(−1 | x) = 0.3 → ideal f(x) = +1, noise level = 0.3

Error Measures

  • 0/1 loss: count misclassifications — used for binary classification
  • Squared error: (ŷ − y)² — used for regression
  • Cross-entropy (log-loss): used for logistic regression

Weighted Classification

Different mistakes can have different costs. Example: false negatives (missing cancer) are far more costly than false positives. We can weight errors asymmetrically in training.

📝 Exam Tip: Understand that the choice of error measure affects what the algorithm optimizes — this is why we use different models for classification vs regression.

Lecture 6
Linear Regression

The Problem

Predict a real-valued output (e.g., credit limit). Use hypothesis h(x) = wᵀx — NO sign function. Error = squared distance between prediction and truth.

In-Sample ErrorEin(w) = (1/N) Σ(wᵀxₙ − yₙ)² = (1/N)||Xw − y||²

The Algorithm — Closed-Form Solution

Unlike PLA, linear regression has an analytic solution. We minimize Ein by taking the gradient and setting it to zero.

Optimal Weights (Closed Form)wLIN = (XᵀX)⁻¹Xᵀy = X†y
where X† is the pseudo-inverse of X

Using Linear Regression for Classification

Use linear regression to get an initial weight vector, then apply sign() for binary classification. Fast way to initialize PLA.

  • Pros: very fast closed-form solution
  • Cons: squared error is a loose upper bound on 0/1 error for large |ys|
📝 Exam Tip: Linear regression = no sign function + minimize squared error + has closed-form solution. Know all three distinguishing facts.

Lecture 7
Logistic Regression

The Problem

Predict the PROBABILITY of an outcome (e.g., P(heart attack | patient features)). Output must be in [0, 1].

Logistic / Sigmoid Functionh(x) = θ(wᵀx) = 1 / (1 + e^(−wᵀx))
θ(0) = 0.5 · Large positive scores → prob near 1 · Large negative → near 0

Error — Cross-Entropy

We use maximum likelihood: find w that makes observed training labels most probable. Maximizing likelihood = minimizing cross-entropy loss.

Cross-Entropy LossEin(w) = (1/N) Σ ln(1 + exp(−yₙ wᵀxₙ))

Gradient Descent Optimization

No closed-form solution! We use gradient descent.

  • η (eta) = learning rate — controls step size
  • Too large η: oscillates, may diverge
  • Too small η: converges very slowly
  • Logistic Ein is smooth and convex → GD guaranteed to find global minimum
📝 Exam Tip: Logistic regression outputs PROBABILITIES (soft classification), not hard {−1, +1} labels. This is its biggest differentiator from the other two linear models.

Linear Models for Classification
The 3 Linear Models Compared
Model Output Error Measure Solution Method
Linear Classification (PLA) sign(wᵀx) → {−1,+1} 0/1 loss Iterative (PLA)
Linear Regression wᵀx → ℝ Squared error Closed-form X†y
Logistic Regression θ(wᵀx) → (0,1) Cross-entropy Gradient Descent

Stochastic Gradient Descent (SGD)

Instead of computing gradient on ALL N points each step, SGD picks ONE random point and updates with its gradient.

  • Full GD: precise direction, expensive per step
  • SGD: noisy direction, cheap per step — often preferred in practice for large datasets

Multiclass Classification

  • One-vs-All (OvA): train K binary classifiers. Assign class with highest confidence score.
  • One-vs-One (OvO): train K(K−1)/2 classifiers for each pair. Use majority vote.
  • Softmax: logistic regression extended to K classes simultaneously
📝 Exam Tip: The key insight — all 3 linear models share the same scoring function s = wᵀx. They differ only in how they interpret and optimize that score.

Lecture 9
Hazard of Overfitting

What is Overfitting?

Ein ↓ (lower training error) BUT Eout ↑ (higher test error). Model memorizes training data instead of learning the underlying pattern.

Contrast with underfitting: both Ein AND Eout are high — model is too simple.

Causes of Overfitting

  • Excessive model complexity (dVC too large relative to N)
  • Stochastic noise: random noise in labels yₙ
  • Deterministic noise: part of f too complex for H to capture
  • Limited data size N

The Bias-Complexity Tradeoff

As model complexity increases:

  • Bias decreases (model can represent more complex functions)
  • Variance increases (model overfits to specific training data)
  • Optimal complexity balances both — neither underfit nor overfit

The Driving Analogy

Overfitting = causing a car accident because you drive too fast on a bumpy road with limited visibility. Solution: slow down (reduce complexity) or get better road info (more data).

📝 Exam Tip: Be able to explain overfitting using the Ein/Eout diagram and name at least 3 causes. This is one of the highest-probability exam topics.

Lecture 10
Regularization

What is Regularization?

A technique to combat overfitting by penalizing model complexity. Instead of just minimizing Ein, we minimize a combined objective.

Augmented ErrorEaug(w) = Ein(w) + (λ/N) · Ω(w)
where Ω(w) is the regularizer (penalty for large weights)

L2 Regularization — Ridge / Weight Decay

L2 RegularizerΩ(w) = ||w||² = Σwᵢ²

Penalizes large weights. Encourages smaller, more spread-out weights.

  • λ large → stronger regularization → simpler model → may underfit
  • λ small → weaker regularization → may overfit
  • λ = 0 → no regularization (same as unregularized)

L1 Regularization — Lasso

L1 RegularizerΩ(w) = ||w||₁ = Σ|wᵢ|

Encourages sparse solutions — many weights go to exactly 0. Useful for feature selection.

Property Ridge (L2) Lasso (L1)
Weight behavior Shrinks all weights equally Zeros out many weights
Feature selection No (keeps all features) Yes (automatic)
Solution Closed-form exists No closed-form (iterative)
Best for When all features matter When only some features matter
📝 Exam Tip: Explain regularization as a "soft constraint" on weights — we're not hard-setting weights to 0, but penalizing large ones. λ controls the tradeoff.

Lecture 11
Validation

The Model Selection Problem

After training multiple models (different λ, polynomial orders), which do we deploy? We cannot use Ein because it always favors the most complex model.

⚠️ Why not use Ein? Selecting by lowest Ein leads to overfitting — complex models always win on training data by memorizing it.

Validation Set Approach

Split data into Dtrain and Dval. Train on Dtrain, evaluate on Dval, pick model with lowest Eval.

  • Typical split: 80/20 or 70/30 train/validation
  • Eval is unbiased estimate of Eout... as long as you don't "peek" at Dval during training
  • After model selection, retrain the chosen model on ALL data (Dtrain + Dval)

Leave-One-Out Cross Validation (LOOCV)

Train N times, each time holding out exactly 1 point. Average the N validation errors.

  • Pros: maximum use of data, nearly unbiased estimate of Eout
  • Cons: computationally expensive (N training runs)

K-Fold Cross Validation

Split into K equal folds. Train K times, each time holding out 1 fold. Average K errors.

  • Common choice: K = 5 or K = 10
  • Balances bias vs. computational cost
  • More practically useful than LOOCV
📝 Exam Tip: Cross-validation is crucial for model selection. Understand WHY we can't use Ein and why we need a held-out set.

Probability & Decision Theory

Bayes' Theorem

  • P(A): prior — what we believe before seeing data
  • P(B|A): likelihood — how likely is evidence B given A is true
  • P(A|B): posterior — updated belief after seeing evidence
Bayes' FormulaP(A|B) = P(B|A) · P(A) / P(B)

Example: P(disease | positive test) = P(positive test | disease) × P(disease) / P(positive test)

Bayes Optimal Classifier

Given class posteriors P(y|x), assign the class with the highest probability. This minimizes the expected 0/1 loss.

Decision Ruleg*(x) = argmax_y P(y|x)

Decision Theory Basics

  • Expected loss / risk: E[loss] = Σ_y P(y|x) · loss(predicted, y)
  • Minimum risk decision: pick the class that minimizes expected loss
  • Asymmetric losses: (e.g., false negative more costly) → decision threshold shifts

Key Probability Concepts

  • Random variable: variable whose value is outcome of a random process
  • Expectation: E[X] = Σ x·P(X=x) (discrete) or ∫ x·f(x)dx (continuous)
  • Variance: Var(X) = E[(X − E[X])²] = E[X²] − (E[X])²
  • Conditional probability: P(A|B) = P(A∩B)/P(B)
  • Independence: P(A∩B) = P(A)·P(B) iff A and B are independent
