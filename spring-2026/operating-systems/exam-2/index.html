Chapter 5
CPU Scheduling
FCFSSJFRound RobinPriority

Must Know High-priority concepts
⚙️
Process vs Program Program = passive (on disk). Process = active (in execution). One program can become multiple processes.
🔄
5 Process States New → Ready → Running → Waiting → Terminated. Only ONE process running per CPU. Many can be Ready or Waiting.
📋
PCB Contents S-P-C-S-M-A-I: State, Program counter, CPU registers, Scheduling info, Memory info, Accounting info, I/O info.
🧵
Thread Benefits (RRES) Responsiveness, Resource sharing, Economy (cheaper than process creation), Scalability (multicore).
🧟
Zombie vs Orphan Zombie = done but parent hasn't called wait(). Orphan = parent died, child still running (re-parented to init).
⚡
SJF is Optimal SJF provably gives minimum average waiting time. But can't know future burst lengths — must estimate with exponential averaging.
😵
Starvation & Aging Priority scheduling can starve low-priority processes. Solution: Aging — gradually increase priority of waiting processes.
📐
Amdahl's Law Speedup = 1/(S + (1-S)/N). Serial portion S limits max speedup as N→∞. Even 25% serial caps speedup at 4×.
